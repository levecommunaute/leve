# LEVE — Cursor Rules
# Plateforme de redistribution communautaire francophone YouTube

## Architecture
Monorepo: apps/web (Next.js 15) + apps/mobile (Expo 52) + packages/shared (business logic)

## Stack
- Frontend Web: Next.js 15 App Router, TypeScript strict, Tailwind CSS, shadcn/ui
- Frontend Mobile: Expo SDK 52, Expo Router, NativeWind
- Backend: Supabase (PostgreSQL + Auth + Edge Functions + RLS)
- Cache: Upstash Redis
- AI: Anthropic Claude API (quiz generation - claude-sonnet-4-20250514)
- Payments: Stripe + PayPal

## Business Rules (CRITICAL - never break these)
- Revenue split: 45% members (PMQ) / 20% production / 10% Foundation / 25% operations
- PMQ formula: value_per_pt = (monthly_revenue × 0.45) / total_pts_pmq
- PCOL: collaborator gets 20% of EARNED points by members (NOT 20% of all available points)
- Members keep 80% of their earned points on collaborator videos
- PA Pool: fixed $5/pt. Tax 2% on ALL PA transactions
- Transfer threshold: $100 minimum solde_banque
- Code fragments: 3 per video, placed at random timestamps
- Multiplicators: pionnier=2.0, fondateur=2.0, communaute=1.0, collaborateur=1.2

## Member Types
1. Pionnier — first 1000 members, multiplier 2.0
2. Fondateur — members 1001-10000, multiplier 2.0
3. Communaute — regular members, multiplier 1.0
4. Collaborateur — content creators, multiplier 1.2 (on external videos)

## Database — 13 Supabase Tables
profiles, videos, codes, code_submissions, quiz_questions, quiz_submissions,
points_transactions, redistribution_history, pa_transactions, pending_pcol,
collaborateurs, banque_leve, foundation_pool

## Code Conventions
- All business logic MUST live in packages/shared/src/ — never in components
- TypeScript strict mode always
- Zod for all API input validation
- All Supabase queries must respect RLS
- Never expose SUPABASE_SERVICE_ROLE_KEY client-side
- Cache Redis: classement TTL 5min, solde 30s, PMQ 1h, codes 30 jours

## File Structure

## Edge Functions (Supabase)
1. redistribution-mensuelle — 1er du mois, calcule 4 pools, credite membres
2. generer-quiz — appelle Claude API, 15 questions QCM par video
3. expire-pending — quotidien 2h, expire Pending > 1 an vers PTC
4. notifier-redistribution — email a tous les membres le jour J
